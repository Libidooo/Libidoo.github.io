# 输出格式规范

## 目录
- [输出状态](#输出状态)
- [格式要求](#格式要求)

## 概览
本文件规定宗派判定前置过滤的唯一输出形式。所有判断结果必须严格按照本格式输出，防止模型"留痕"或生成中间过渡状态。

## 核心内容

### 输出状态

#### ✅ 允许进入分析

```
状态: 进入分析
说明: 条目被视为需要进一步解释与判定的分析对象
```

**含义**：
- 条目通过所有过滤规则
- 允许进入下一步系统（主体判定、宗派判定、结构分析等）
- 视为"分析对象"而非"资料占位符"

#### ❌ 直接丢弃

```
状态: 丢弃
说明: 条目被标注或判定为不进入分析范围
```

**含义**：
- 条目不满足进入分析的条件
- 必须直接从数据集中移除
- 不得出现在任何分析结果、候选集或中间输出中

### 格式要求

#### 唯一输出原则
- 每个条目的判断结果仅输出两种状态之一
- 禁止生成中间过渡状态（如"待定""部分通过""需人工复核"等）
- 禁止输出不确定的判断结果

#### 输出内容
- 必须包含：状态（进入分析 / 丢弃）
- 建议包含：说明（简要说明判定理由）
- 禁止包含：被丢弃条目的详细内容（除非用于调试目的）

#### 批量处理输出格式
当处理多个条目时，采用以下格式：

```
条目1: 状态: 丢弃, 说明: 显性备注型丢弃（包含"不抽取实体"）
条目2: 状态: 进入分析, 说明: 明确呈现宗教叙述
条目3: 状态: 丢弃, 说明: 功能性记录文本（施主名单）
...
```

#### JSON格式（可选）
如需结构化输出，采用以下格式：

```json
{
  "条目ID": "entry_001",
  "状态": "丢弃",
  "说明": "显性备注型丢弃（包含'不抽取实体'）",
  "触发规则": "R1"
}
```

## 验证规则

### 输出检查清单
- [ ] 每个条目仅输出两种状态之一
- [ ] 无中间过渡状态
- [ ] 被丢弃条目标记清晰
- [ ] 允许进入分析条目标记清晰
- [ ] 说明简洁明确，指出判定理由

### 常见错误
- ❌ 错误：输出"待定"状态
- ❌ 错误：输出"部分通过"状态
- ❌ 错误：在被丢弃条目后保留详细内容
- ❌ 错误：输出多个状态组合

### 正确示例
```
示例1（正确）:
条目1: 状态: 丢弃, 说明: 显性备注型丢弃（包含"不抽取实体"）

示例2（正确）:
条目2: 状态: 进入分析, 说明: 明确呈现宗教叙述

示例3（错误）:
条目3: 状态: 待定, 说明: 需要进一步人工复核  # 错误：禁止中间状态
```

## 约束与注意事项
- 输出格式必须严格遵守，禁止根据任务需求灵活调整
- 状态字段必须准确，不能模糊表述
- 说明字段必须清晰，指出具体判定理由和触发规则
- 批量处理时保持格式一致性
