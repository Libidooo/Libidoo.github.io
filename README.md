## 系统简介

本系统用于系统化分析四川地区佛教石窟铭文（石刻题记、造像记、碑刻等），自动判定其所属宗派体系。系统采用**符号推理引擎 + AI 协同**的神经符号架构：引擎负责词条匹配、权重计算和冲突消解，AI 负责结果验证、语境补充和不可判定案例的独立判断，基于神经符号AI范式实现低资源条件下的宗派判定。

### 核心能力

| 能力 | 说明 |
|------|------|
| 宗派判定 | 将铭文分类到十六卷宗派体系（净土/密教/华严/禅宗/三阶教/三论/唯识/天台/成实/律宗 + 观音/地藏/弥勒/药师主体卷 + 区域卷 + 经文词干卷） |
| 多字段输入 | 支持铭文文本 + 造像题材 + 年代 + 地点四字段联合判定 |
| 异体字归一 | 自动将繁体/异体字转换为标准字形（藥→药、瑠→璃、來→来等） |
| 复合词保护 | 自动合并相邻短词条，防止跨卷拆分冲突（"孔雀"+"明王"→"孔雀明王"） |
| 互斥消解 | 互斥矩阵自动处理跨宗派冲突，密教采用协同机制（W≥60主判卷+S≥40协同卷） |
| 观音子分类 | 自动检测观音体系内部子分类（天台系/心经系/密教系/民间大乘） |
| CBETA 查询 | 集成 54 部大藏经经典的 T 号/经名/关键词/卷别四级索引查询 |
| 自我进化 | 在处理新铭文时识别新关键词并提示研究者审核加入词库 |

## 快速开始

### 环境要求

- Python 3.8+
- 依赖包见 [requirements.txt](requirements.txt)

### 安装

```bash
git clone https://github.com/Libidooo/Buddhism_sect_Judegment_Skill.git
cd buddhist-judgment
pip install -r requirements.txt
```

### 30 秒上手

```bash
# 单条铭文判定
python scripts/sect_engine.py --input "弟子某甲为亡父造药师如来像一躯愿亡父往生净土"

# 多字段输入（铭文 + 造像题材 + 年代 + 地点）
python scripts/sect_engine.py \
  --input "敬造药师琉璃光如来一躯" \
  --subject "药师佛" \
  --date "乾德三年" \
  --location "安岳"

# 运行内置测试
python scripts/sect_engine.py --test

# JSON 格式输出
python scripts/sect_engine.py --input "敬造药师琉璃光如来一躯" --json
```

### CLI 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | string | `""` | 铭文文本，宗派判定的主要依据 |
| `--subject` | string | `""` | 造像题材，与铭文拼接后统一匹配 |
| `--date` | string | `""` | 造像年代，仅展示，不参与匹配 |
| `--location` | string | `""` | 造像地点，仅展示，不参与匹配 |
| `--json` | flag | — | 以 JSON 格式输出结果 |
| `--test` | flag | — | 运行内置测试用例 |

### 示例 2：密教协同机制

密教W≥60时为主判卷，其他S≥40的卷作为协同卷保留，反映唐代以降显密交融的历史事实：

```bash
python scripts/sect_engine.py --input "造药师琉璃光如来大日如来曼荼罗灌顶"
# → 主判卷：密教（卷5），协同卷：药师（卷4）
```

### 示例 3：复合词保护

"孔雀明王"由"孔雀"（卷3净土, W=45）和"明王"（卷5密教, W=54）两个短词组成。引擎自动检测相邻短词并合并为复合词条"孔雀明王"（卷5密教, W=80），防止错误拆分。

```bash
python scripts/sect_engine.py --input "敬造孔雀明王像一躯"
# → 判定为密教体系（卷5），S=80
```

```bash
python scripts/sect_engine.py --input "观世音菩萨普门品愿往生西方极乐世界"

```

### 示例 5：异体字归一化

```bash
python scripts/sect_engine.py --input "敬造藥師琉瑠光如來一軀"
```

### S 得分计算公式

```
S = Σ(W≥60的权重) + Σ(0.5 × W<60的权重)
```

**权重等级**：

| 等级 | 权重范围 | 说明 |
|------|----------|------|
| A层 决定因 | W=80-85 | 主体明确，决定性证据 |
| B层 强因缘 | W=60-79 | 强力证据，显著支持判定 |
| C层 中因缘 | W=40-59 | 中等证据，辅助性支持 |
| D层 弱因缘 | W=20-39 | 弱证据，仅作参考 |

**判定阈值**：

| S 得分 | 判定等级 | 说明 |
|--------|----------|------|
| S ≥ 160 | 确立（Established） | 判定明确，无需进一步检查 |
| 120 ≤ S < 160 | 高可信（Confident） | 判定可靠，需检查冲突卷 |
| 80 ≤ S < 120 | 弱可信（Weak） | 判定较弱，需补强 |
| S < 80 | 不可判定 | 需人工复核 |

> **绝对证据"一票确立"规则**：W≥80的词条不受S≥160阈值限制，单条即可判定为"确立"。

### 优先级规则

1. **密教协同机制**：密教W≥60时为主判卷，其他S≥40的卷作为协同卷保留（反映唐代以降显密交融的历史事实）
2. **主体优先级**：基于宗派信号可信度排序——弥勒→阿弥陀→观音→地藏→药师（信号越强优先级越高）
3. **互斥矩阵消解**：跨卷冲突由互斥矩阵自动消解（如观音与地藏绝对互斥、弥勒与地藏绝对互斥）
4. **复合词保护**：相邻短词合并为复合词，防止跨卷拆分

## 引擎与 AI 协作

本系统的核心设计原则：**权重计算必须由确定性引擎完成，AI 不得心算**。

### 分工边界

| 任务 | 执行者 | 原因 |
|------|--------|------|
| 异体字归一化 | 引擎 | 确定性映射，无需判断 |
| 词条匹配 | 引擎 | 最长匹配算法，避免遗漏 |
| S 得分计算 | 引擎 | 数学公式，禁止心算 |
| 互斥冲突消解 | 引擎 | 规则化优先级，无歧义 |
| 结果验证 | AI | 需要语义理解和上下文 |
| 不可判定案例判断 | AI | 引擎信号不足时独立判断 |
| 误判纠正 | AI | 基于整体语境纠正局部误判 |
| 最终报告 | AI | 需要自然语言生成与可解释性 |

### 协作流程

```
用户输入 → 引擎计算 → AI协同判断 → 标准化输出
                         ↑
                    引擎输出结构化结果
                    (S值/互斥判定/不可判定标记)
                    AI 三种策略：
                    A. 验证确认（引擎正确）
                    B. 独立判断（引擎不可判定）
                    C. 语境纠正（引擎误判）
```

### 三类AI辅助模式

| 模式 | 描述 | 案例数 | AI F1 |
|------|------|--------|-------|
| A类：引擎正确→AI验证 | 引擎已正确，AI确认并增强可解释性 | 169 | 0.9567 |
| B类：引擎不可判定→AI新判 | 引擎信号不足，AI基于语境独立判断 | 129 | 0.8425 |
| C类：引擎误判→AI纠正 | 引擎纯关键词误判，AI基于语境纠正 | 114 | 0.8089 |
