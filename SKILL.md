# 四川地区佛教石窟铭文宗派判定系统

## 系统概述

本项目包含三个 Skill，用于系统化分析四川地区佛教石窟铭文内容并判定主体与宗派：

1. **buddhist-inscription-judgment-system**（总控 Skill）
2. **sect-preliminary-filter**（前置过滤 Skill）
3. **sect-judgment-hub**（宗派判定主控 Skill）

**适用范围**：四川地区佛教石窟铭文（石刻题记、造像记、碑刻等）

## 计算引擎

本系统集成确定性计算引擎 `sect_engine.py`，用于替代 AI 心算，防止权重计算幻觉。

### 引擎位置

```
scripts/
├── sect_engine.py          # 宗派判定计算引擎主程序
└── data/
    ├── term_index.json     # 词条索引（8100+ 词条）
    ├── mutex_matrix.json   # 6×6 基础互斥矩阵
    ├── extended_mutex.json # 10×10 扩展互斥矩阵
    └── ...                 # 权重定义、S得分阈值等
```

### 引擎调用方式

**当需要对铭文进行宗派判定时，必须调用计算引擎**：

```bash
# 单条铭文判定（仅铭文文本）
python scripts/sect_engine.py --input "铭文内容"

# 多字段输入（铭文 + 造像题材 + 年代 + 地点）
python scripts/sect_engine.py --input "铭文内容" --subject "造像题材" --date "年代" --location "地点"

# 仅造像题材判定（无铭文时）
python scripts/sect_engine.py --subject "造像题材"

# 运行内置测试用例
python scripts/sect_engine.py --test

# JSON 格式输出（适合程序化处理）
python scripts/sect_engine.py --input "铭文内容" --subject "造像题材" --json
```

**参数说明**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | 字符串 | `""` | 铭文文本，宗派判定的主要依据 |
| `--subject` | 字符串 | `""` | 造像题材，与铭文拼接后统一匹配，记录字段归属 |
| `--date` | 字符串 | `""` | 造像年代，仅在报告中展示，不参与词条匹配 |
| `--location` | 字符串 | `""` | 造像地点，仅在报告中展示，不参与词条匹配 |
| `--json` | 标志 | — | 以 JSON 格式输出结果 |
| `--test` | 标志 | — | 运行内置测试用例 |

### 引擎功能

1. **异体字归一化**：将铭文中的异体字转换为标准字形
2. **词条匹配**：精确最长匹配 + 全量扫描（8100+ 词条库）
3. **复合词保护**：自动合并相邻的短词条（≤3 字），当拼接结果存在于词条索引中时，合并为复合词条，防止跨卷拆分冲突（如"孔雀"+"明王"→"孔雀明王"归卷5密教）
4. **S 得分计算**：`S = Σ(W≥60) + Σ(0.5 × W<60)`
5. **互斥冲突消解**：密教绝对优先 → 主体优先级 → 互斥矩阵
6. **卷2观音子分类**：自动检测观音体系内部子分类（天台系/心经系/密教系/民间大乘），在判定宗派中附加显示
7. **字段贡献分析**：当同时输入铭文和造像题材时，分别计算各字段对判定结果的 S 得分贡献
8. **标准化报告生成**：输出【铭文】【判定宗派】【S得分】【判定等级】【证据链】【各卷得分汇总】【字段贡献】

### 引擎与 AI 协作规则

- **权重计算必须使用引擎**：AI 不得自行心算 S 得分，必须调用 `sect_engine.py`
- **词条匹配必须使用引擎**：AI 不得自行在词条库中搜索匹配，由引擎自动完成
- **AI 负责解释结果**：引擎输出数值结果后，AI 负责解读证据链、补充上下文、生成最终判定
- **引擎无法处理的情况**：铭文残损严重、词条库未覆盖的新术语 → AI 可补充判断并标注"引擎未覆盖"

## 执行流程

### 步骤一：前置过滤

加载 `sect-preliminary-filter` Skill，过滤无效题记。

**Skill功能**：
- 过滤无效题记（无关内容、现代添加、伪造铭文等）
- 识别非佛教内容
- 检测残损过度的题记
- 标记允许进入分析的条目

### 步骤二：计算引擎判定

**调用计算引擎**对通过前置过滤的铭文进行自动化判定：

```bash
python scripts/sect_engine.py --input "铭文内容"
```

引擎自动完成：异体字归一化 → 词条匹配 → S得分计算 → 互斥消解 → 生成报告

### 步骤三：AI 解读与补充

AI 基于引擎输出进行解读：
- 验证引擎判定结果是否合理
- 补充引擎未覆盖的上下文信息（如图像描述、历史背景）
- 对低分或不可判定结果给出补充分析
- 生成最终的标准化输出

**Skill功能**：
- 卷零总则应用：理解全局框架、互斥矩阵、权重体系
- 卷一至卷十五完整判定
- 证据权重计算：由引擎自动完成（A层决定因、B层强因缘、C层中因缘、D层弱因缘）
- 卷间交叉逻辑处理：由引擎自动完成（互斥矩阵、主体优先级、冲突消解）
- 输出标准化结果：宗派判定、信任得分、证据链、备注

## 核心公式与定义

### 综合得分（S）计算公式

**权重分（W）与综合得分（S）的转换公式**：

```
S = ΣW主要证据i + Σ(0.5 × W次要证据i)
```

其中：
- `W主要证据i` = 第 i 个主要证据的权重分
- `W次要证据i` = 第 i 个次要证据的权重分
- `S` = 综合得分（主要证据原数计入 + 次要证据减半计入）

**权重分等级定义**：
- **A层决定因**：W = 80-85分（主体明确，决定性证据）
- **B层强因缘**：W = 60-79分（强力证据，显著支持判定）
- **C层中因缘**：W = 40-59分（中等证据，辅助性支持）
- **D层弱因缘**：W = 20-39分（弱证据，仅作参考）

**综合得分与判定结果对应表**：
| 综合得分（S） | 判定结果 | 说明 |
|--------------|---------|------|
| S ≥ 160 | 确认为该卷 | 判定结果明确，无需进一步检查 |
| 120 ≤ S < 160 | 高概率 | 判定结果可靠，需检查冲突卷 |
| 80 ≤ S < 120 | 弱概率 | 判定结果较弱，需主体补强 |
| S < 80 | 不可判定 | 判定结果不可靠，需人工复核 |

**示例计算**：
```
题记："弟子某甲，为亡父造药师如来像一躯，愿亡父往生净土，安嶽工匠文氏造"

卷四药师判定：
- 主要证据：
  - 药师如来：W = 85（A层决定因）
  - 药师像：W = 75（B层强因缘）
- 次要证据：
  - 往生净土：W = 60（B层强因缘，次要证据）
  - 安嶽工匠文氏：W = 45（C层中因缘）
- 计算：
  - S主要 = 85 + 75 = 160
  - S次要 = (0.5 × 60) + (0.5 × 45) = 30 + 22.5 = 52.5
  - S = 160 + 52.5 = 212.5
- 判定结果：确认为该卷（S ≥ 160）
```

### 主要证据与次要证据定义

**主要证据（Primary Evidence）**：
- **定义**：决定题记宗派取向的直接证据，权重 1:1 原数计入
- **特征**：
  - 直接指向宗派或主体
  - 具有决定性或强有力的支持作用
  - 通常是专有名词、核心概念或明确表述
- **包含内容**：
  1. **铭文中的宗派判定词**：如"生西方""往生""十二大愿""佛顶尊胜陀罗尼""无量光佛"等
  2. **铭文中出现的经文原句/经文词干**：如"观世音菩萨普门品""药师琉璃光如来本愿功德经"等
  3. **铭文中出现的本尊名号**：如阿弥陀佛、药师佛、尊胜幢、大威德等
  4. **古代原刻题名**：题名若来自古代原刻（非现代研究命名），视为主要证据
  5. **宗派决定因图像**：图像若为宗派决定因（如千手千眼、药师净土三重结构），可计入主要证据
- **示例**：
  - A层：药师如来（W=85）、阿弥陀佛（W=85）、观世音菩萨（W=85）、佛顶尊胜陀罗尼（W=85）
  - B层：药师像（W=75）、往生（W=75）、十二大愿（W=75）、千手千眼（W=75）

**次要证据（Secondary Evidence）**：
- **定义**：仅具有"辅助性"的证据，权重减半计入（0.5 × W）
- **特征**：
  - 间接支持宗派或主体判定
  - 具有辅助性或参考性作用
  - 通常是描述性词汇、辅助概念或模糊表述
- **包含内容**：
  1. **地点信息**：如"安岳、简州"为药师/密教高频区
  2. **特定时代宗派流行趋势**：如广政年间密教高频
  3. **匠人、施主身份**：不构成宗派因缘，仅提示语境
  4. **图像构图的一般因缘**：如"三世佛""一佛二菩萨"等通用图像
  5. **杜撰性较高的题名**：现代命名、含糊的标签
- **示例**：
  - C层：造像（W=45）、供养（W=45）、发愿（W=45）、安嶽（W=45）
  - D层：弟子某甲（W=30）、某年某月（W=25）、工匠姓名（W=30）

**证据分级判定规则**：

1. **主要证据判定（直接证据）**：
   - 铭文中的宗派判定词（如：生西方、往生、十二大愿）
   - 经文原句/经文词干（如：观世音菩萨普门品）
   - 本尊名号（如：阿弥陀佛、药师佛、尊胜幢）
   - 古代原刻题名
   - 宗派决定因图像（如：千手千眼、药师净土三重结构）

2. **次要证据判定（辅助证据）**：
   - 地点信息（如：安岳、简州）
   - 特定时代宗派流行趋势（如：广政年间密教高频）
   - 匠人、施主身份
   - 图像构图的一般因缘（如：三世佛、一佛二菩萨）
   - 杜撰性较高的题名（现代命名）

3. **综合得分判定规则**：
   - S ≥ 160：确认为该卷（主要证据充分）
   - 120 ≤ S < 160：高概率（主要证据较多，需检查冲突卷）
   - 80 ≤ S < 120：弱概率（主要证据不足，需主体补强）
   - S < 80：不可判定（主要证据缺失，需人工复核）

4. **证据等级调整规则**：
   - 主要证据 ≥ 2 个 → S ≥ 160，确认为该卷
   - 主要证据 = 1 个 + 次要证据 ≥ 3 个 → 120 ≤ S < 160，高概率
   - 主要证据 = 0 个 + 次要证据 ≥ 5 个 → S < 80，不可判定，需人工复核
   - 证据之间存在冲突 → 应用互斥矩阵消解

**输出格式示例**：
```
线索：
- 主要证据（原数计入）：
  - 卷四·A层决定因：药师如来（W=85）
  - 卷四·B层强因缘：药师像（W=75）
  - 卷四·B层强因缘：往生净土（W=60）
- 次要证据（减半计入）：
  - 卷八·C层中因缘：安嶽工匠文氏（W=45×0.5=22.5）

计算：
- S主要 = 85 + 75 + 60 = 220
- S次要 = 22.5
- S = 220 + 22.5 = 242.5

判定结果：确认为该卷（S ≥ 160）
综合得分：S = 242.5
备注：主要证据充分，判定结果明确
```


## 注意事项

1. **前置过滤绝对优先**：任何条目必须先通过前置过滤，未通过者不得进入宗派判定
2. **主体判定优先**：五大主体（药师/弥勒/地藏/观音/阿弥陀）一旦判定，不得跨卷判断宗派
3. **密教绝对优先**：出现W≥60的密教词（种子字/手印/曼荼罗/大日如来/明王）→必入卷五，其他卷全部靠后。此规则由 `sect_engine.py` 的 `MutexResolver` 自动强制执行（`ESOTERIC_THRESHOLD = 60`），AI 无需手动介入
4. **异体字工具化**：异体字仅用于题记识别和OCR校读，不得影响宗派权重
5. **四川地域特色**：重视卷八区域体系的地域信息，正确识别四川地名异体字
